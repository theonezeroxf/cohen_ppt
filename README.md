# PPT Batch Split, Template Strategy, Image Background & Content Skill

> 使用 **`img-gen + Presentation`** 将 PDF 指定内容规划并生成 PPT：生成前先确认 PPT 制作思路（按 PDF 内容思路或输入自定义思路），并在 **风格样式描述 / logo 风格模板 / 复用模板库 PPT** 中三选一作为唯一视觉来源；生成的新模板必须保存为可复用 `.pptx`，方便后续复用和导出；`img-gen` 负责生成背景图，以及按 `Presentation` 的要求生成所需图片；其余 PPT 构建、排版、备注、分批和合并工作全部交给 `Presentation`。

## 项目定位

本项目是一个 Codex skill，适用于从 PDF 生成 PPT 的场景，例如：

- 论文答辩 / 学术汇报；
- 项目报告 / 商业提案；
- 教学课件 / 培训材料；
- 研究总结 / 技术方案说明。

它的目标不是把 PDF 页面简单截图成 PPT，而是梳理 PDF 主要内容，同时通过三选一的视觉来源（风格样式描述、logo 驱动的新模板、模板库复用）统一视觉系统，避免重复询问或叠加冲突，并使用生成式背景提升视觉效果。

## 实现技术

项目实现技术为 **`img-gen + Presentation`**：

| 技术 | 职责 |
| --- | --- |
| `img-gen` | 生成每页背景图；为风格样式或 logo 驱动的新模板生成背景、装饰 motif，或按 `Presentation` 的版式、尺寸、安全区和视觉需求生成必要图片素材。 |
| `Presentation` | 负责除图片生成外的其余工作：三选一视觉来源解析、模板库 PPT 复用、风格样式模板或 logo/prompt 模板系统创建、生成模板 `.pptx` 保存、规划落地、创建 PPT、排版、插入图片、添加文本/形状/表格、添加可选备注、分批组装和合并。 |

关键规则：生成 PPT 前必须先确认制作思路和唯一视觉来源：制作思路可按 PDF 内容自动梳理，或采用用户输入的自定义思路；视觉来源必须在风格样式描述、logo 风格模板、复用模板库 PPT 中三选一。选择 logo 或模板库时，不再额外询问独立风格描述；选择风格样式描述时，才使用用户风格或默认学院简约风格生成模板。`img-gen` 只产出背景图、模板装饰元素和 `Presentation` 要求的图片素材；标题、要点、结论、重要标签、主要 PDF 摘要等关键文本，以及模板应用、版式组织、备注、页码和最终 PPT 结构，必须由 `Presentation` 处理。

## 仓库文件

- `SKILL.md`：Codex 执行该 skill 时使用的完整规则，包含风格样式描述、模板库复用与 logo/prompt 新模板生成三选一流程。
- `tools/batch_plan_template.json`：生成 PPT 前使用的结构化计划模板，用于记录预检问题、页数分配、大纲、逐页背景 prompt、分批计划、合并计划和质量检查。

## 执行前必须询问的问题

在分析 PDF、生成大纲、选择/生成模板、调用 `img-gen` 或创建 PPT 之前，必须先确认以下问题；如果用户请求中已经明确给出，则无需重复询问。

1. **PDF 中哪部分内容要生成 PPT？**
   - 可接受：页码范围、章节、节标题、图表范围、指定主题等。
   - 如果 PDF 范围未知，不能继续执行。
2. **PPT 制作思路采用哪一种？**
   - 按 PDF 内容思路：由系统根据 PDF 范围、内容密度、重要性和叙事优先级梳理汇报逻辑。
   - 输入自定义思路：要求用户提供自定义叙事、受众、重点、结构或具体制作要求，并在大纲和排版中优先遵循。
3. **生成 PPT 前采用哪一种视觉来源？必须三选一，不能重复叠加。**
   - 风格样式描述：要求用户描述风格、情绪、配色、受众和限制；若未指定，默认使用 **学院简约风格 / academic minimalist style**，并由 `Presentation` 生成可复用模板。
   - logo 风格模板：要求用户提供 logo 文件/路径/图片引用，以及描述品牌、受众、情绪、配色和限制条件的模板提示词；logo 与提示词就是风格来源，不再另问风格。
   - 复用模板库 PPT：要求用户选择或描述模板名称、路径、分类、缩略图或偏好；该模板就是风格来源，不再另问风格。
   - 通过风格样式或 logo 新生成的模板必须保存为 `.pptx`，方便之后复用和导出。
4. **是否为 PPT 生成备注？**
   - 如果需要：为每页生成简洁演讲者备注。
   - 如果不需要：不添加备注。
   - 如果未指定且必须继续：默认不生成备注，并在计划中记录该默认值。

## 标准工作流

1. 确认 PDF 范围、PPT 制作思路、唯一视觉来源/模板策略、是否生成备注。
2. 先处理 PPT 制作思路：
   - 若选择按 PDF 内容思路，根据 PDF 范围、内容密度、重要性和叙事优先级自动组织汇报逻辑。
   - 若选择输入自定义思路，将用户提供的叙事、受众、重点、结构或制作要求写入计划，并用于页数分配、大纲和视觉层级。
3. 再处理唯一视觉来源/模板策略：
   - 若选择风格样式描述，按用户风格或默认学院简约风格生成封面、目录、章节页、内容页、图表页、总结页和结束页等可复用布局，并保存为模板 `.pptx`。
   - 若选择复用模板库 PPT，读取/选择对应模板，并保留主题色、字体、母版布局、章节页、占位符和页码风格。
   - 若根据 logo 和提示词生成新模板，先分析 logo 配色、几何、对比度和品牌气质，结合提示词生成封面、目录、章节页、内容页、图表页、总结页和结束页等可复用布局，并保存为模板 `.pptx`。
   - 模板必须由 `Presentation` 应用为统一布局系统。
4. 分析指定 PDF 内容：
   - 内容量 / 密度；
   - 概念重要程度；
   - 汇报叙事优先级；
   - 图、表、图表、公式、架构图的重要性；
   - 是否需要 1 页、多页，或只需简要提及。
5. 按内容量和重要程度分配 PPT 页数，不能按 PDF 页数平均分配。
6. 生成完整 `slide_outline`。
7. 如果计划超过 10 页，生成 `batch_plan`，按逻辑章节拆成 5–10 页一批。
8. 由 `Presentation` 确定每页图片需求、尺寸、风格和安全区域，再调用 `img-gen` 生成背景图或必要图片素材。
9. 使用 `Presentation` 插入每页背景图和图片素材。
10. 使用 `Presentation` 完成排版，并添加标题、要点、结论、说明、图表注释等 PowerPoint 内容。
11. 仅在用户要求时使用 `Presentation` 添加备注。
12. 执行最终质量检查。

## 页数分配建议

如果用户未指定总页数，可参考以下默认分配：

| 内容区域 | 建议页数 |
| --- | ---: |
| 封面 / 标题 | 1 |
| 目录 / 汇报结构 | 1（需要时） |
| 背景 / 动机 | 1–3 |
| 问题定义 / 目标 | 1–2 |
| 相关工作 / 上下文 | 1–3 |
| 方法 / 设计 / 框架 | 2–5 |
| 数据 / 实验设置 / 实现 | 1–3 |
| 结果 / 分析 / 讨论 | 2–6 |
| 案例 / 应用 | 1–3 |
| 总结 / 展望 | 1–2 |
| 致谢 / Q&A | 1（需要时） |

## `img-gen` 背景要求

`img-gen` 的输入应由 `Presentation` 的页面规划驱动。每页背景或图片素材 prompt 应包含：

- 页码和标题；
- 当前页的目的 / 核心信息；
- 已解析的唯一视觉来源；仅当选择风格样式描述且未指定风格时使用 **学院简约风格**；
- 配色方案；
- `Presentation` 指定的版式方向、图片尺寸和正文安全区域；
- 背景类型：抽象、学院、商务、技术、极简、插画等；
- 可参考的 PDF 图像区域；
- 避免背景图干扰正文阅读的说明。

除背景图和 `Presentation` 明确要求的图片素材外，`img-gen` 不负责 PPT 的文字排版、备注、页码、分批、合并或最终结构。

## 10 页以上 PPT 的分批策略

当计划 PPT 超过 10 页时：

1. 先分析完整 PDF 指定范围。
2. 先生成完整页数分配和 `slide_outline`。
3. 按逻辑章节拆分批次，而不是机械按页码切分。
4. 每批 5–10 页；设计复杂时优先 5 页，普通学术/报告类优先 6–8 页。
5. 保持统一的主题、字体、页码、背景风格和内容叠加规则。
6. 优先通过 `Presentation` 基于统一 slide specification 生成最终 PPT；如果生成了独立批次文件，再通过 `Presentation` 进行组装或合并。

## 使用 `batch_plan_template.json`

生成 PPT 前，使用 `tools/batch_plan_template.json` 作为计划脚手架，至少填写：

- `preflight_questions`：PDF 范围、PPT 制作思路、三选一视觉来源/模板策略、备注偏好；
- `deck_plan`：目标页数、实际计划页数、制作思路、三选一视觉来源、模板 `.pptx` 路径、风格系统；
- `slide_count_allocation`：每个 PDF 部分的内容量、重要性、视觉复杂度、分配页数和原因；
- `slide_outline`：每页标题、来源、核心信息、`img-gen` 背景 prompt、备注状态；
- `batch_plan`：超过 10 页时的批次范围和批次检查；
- `merge_plan`：是否需要合并、最终输出路径和合并检查；
- `quality_checks`：交付前检查项。

## 最终质量检查

交付前确认：

- PDF 范围已确认；
- 已在风格样式描述、logo 风格模板、复用模板库 PPT 中确认唯一视觉来源；
- 通过风格样式或 logo 新生成的模板已保存为 `.pptx`，以便复用和导出；
- 是否生成备注已确认，或已默认不生成；
- 页数符合计划或用户要求；
- 页数分配体现内容量和重要程度；
- 每页顺序匹配大纲；
- 每页有 `img-gen` 背景图，或有按 `Presentation` 要求生成/选择的图片素材；
- 备注只在用户要求时存在；
- 页码连续；
- 批次之间风格一致；
- 背景留有足够对比度和空白区域承载正文。
